THE TECHNICAL VERSION (5-MIN READ)
================================================================================

Architecture:
  SimulatedWorld
    ├── TimeEngine (deterministic clock - same inputs = same results)
    ├── HTTPInterceptor (mocks Calendar/Slack APIs)
    ├── MutationLog (tracks all data changes)
    ├── Tracer (logs all requests/responses)
    └── Integrations (Calendar, Slack, Gmail, etc.)

Flow:
  1. SETUP    → Create users, calendars, conflicts (3 users, 3 TZ)
  2. EXECUTE  → Run meeting scheduling action (make API calls)
  3. OBSERVE  → Capture mutations & traces (full audit trail)
  4. EVALUATE → Score against rubric (3 targets × 3-4 criteria)
  5. REPORT   → Identify gaps & guide improvements

Evaluation:
  • 3 target dimensions (Availability, Creation, Experience)
  • 10 weighted criteria (0-100 score each)
  • Automatic aggregation to overall score
  • Pass/fail threshold (70%)

Key Design:
  • Deterministic (time is controlled, not flowing)
  • Observable (all mutations and traces captured)
  • Measurable (numeric scores, not subjective)
  • Extensible (easy to add scenarios, integrations, criteria)
  • Reproducible (same scenario = same results)

Demo Results:
  BASELINE:  78.8% = (75% + 100% + 61.5%) / 3, weighted
  IMPROVED:  90.0% = (80% + 100% + 90%) / 3, weighted
